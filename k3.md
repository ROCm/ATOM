## 1. 模型结构

### 1.1 逐层混合骨架
```
KimiK3ForCausalLM → language_model
  ├─ embed_tokens
  ├─ layers[~95]: KimiDecoderLayer
  │    Attention: layer∈kda_layers → KimiKDAAttention(线性)   else → KimiFullAttention(MLA)
  │    FFN:       layer≥first_k_dense_replace → KimiSparseMoeBlock   else → KimiMLP(dense, layer0)
  ├─ norm + [output_attn_res_*]
  └─ lm_head
```

### 1.2 KDA 线性注意力(71 层)
门控 delta-rule 线性注意力(细粒度门变体)。5 路进核 + 1 路输出门:
```
q/k/v_proj ─cat→ causal_conv1d(+silu) → q,k,v   (读写 conv_state)
f_a→f_b_proj → 门 g     b_proj → beta     g_proj → 输出门
```
递归核 `_run_kda`(外部 `fla.ops.kda`),每 token 维护 `S[dv,dk]` 关联记忆(= `ssm_state`,fp32):
```
q,k=L2norm ; alpha=f(g,A_log,dt_bias)逐通道遗忘 ; b=sigmoid(beta)
S = S*alpha            # ④ 遗忘
v_old = S@k ; e = v-v_old
S = S + b*outer(e,k)   # ⑦ delta:擦旧写新
o = S@q                # ⑧ 输出
```
cache 定长 O(1):`conv_state`(bf16 滑窗)+ `ssm_state`(fp32 记忆矩阵)。

### 1.3 MLA 全注意力(24 层,head_dim=192)
DeepSeek-MLA 低秩投影,但 `use_mla=False`,**V padding 到 192 存标准 paged-MHA cache**。MQA 式 k_rope;
RoPE 仅作用 64 维 qk_rope;**输出 sigmoid 门** `attn_out*=sigmoid(g_proj(x))`。head_dim=192 见 §4。

### 1.4 MoE(`KimiSparseMoeBlock`)
DeepSeek-V3 路由(sigmoid + correction bias + grouped top-k)+ 可选 **latent 压缩**(专家在 `Rh` 隐维算,
down/up 全专家共享)+ **共享专家**(N 个融合成一个胖 MLP)+ **SiTU 激活**。
latent 路径 **all-reduce 在 norm 之前**(commit e68fa86)。

### 1.5 Attn-Residual(块残差,每 12 层)
沿深度方向的动态残差:维护快照栈,块内累加、块边界存快照并重置;每层 attn/mlp 前对
`[全部快照 + 当前流]` 做逐 token softmax 加权检索(`_apply_attn_res`),出网络再检索一次。

### 1.6 SiTU 激活
```
out = beta·tanh(gate/beta)·sigmoid(gate) · linear_beta·tanh(up/linear_beta)   (beta=4, linear_beta=25)
```
tanh 软钳制的有界 SiLU;`silu` 顶替会污染大幅值输出(曾致掉点)。

## 3. ★ gfx950 性能重点(还要做什么)

按收益排序。核心矛盾:**gfx1250 靠 flydsl 融合 kernel 解决的东西,gfx950 都还没有,现在靠 triton + torch 兜底撑着。**

### P0 — MoE 相关(92 层,decode 主要开销)

1. **★★★ 融合 SiTU 进 triton MoE kernel**
   现状:MoE 专家两个 GEMM 之间,situ 用 **torch fp32** 现算(`raw_intermediate` 落 HBM fp32、再读回)。
   92 个 MoE 层每层每 token 一次,是 gfx950 MoE 的**最大浪费**。
   要做:在 aiter triton MoE(`moe_op_gemm_a16w4` / `fused_clamp_act_mul`)里加 **in-kernel situ**
   (参考公式 `beta·tanh(g/beta)·sigmoid(g)·lbeta·tanh(up/lbeta)`,beta/lbeta 传入或编译期常量),
   把 stage1→act→stage2 融成不落地。**这是 gfx950 MoE 的头号性能项。**

2. **★★★ 拆掉 MoE 路由的 torch 兜底**
   现状:`AITER_USE_TORCH_TOPK=1`,路由 topk 跑在 torch(调试兜底,慢)。
   要做:让 aiter 的 `biased_grouped_topk`(sigmoid+bias+grouped,asm)在 gfx950 上跑对,去掉 torch 兜底。
   每个 MoE 层都过路由,收益覆盖全模型。

3. **★★ MoE 激活量化 a16w4 → a8w4 / a4w4**
   现状:gfx950 用 `moe_gemm_a16w4`(**bf16 激活**),没吃到低精度激活的带宽/算力收益。
   要做:启用 a8w4/a4w4 激活量化的 MoE + **配套 situ**(注意 a8w4 GGUU decode 目前 silu-only、assert 拒绝 situ,
   需要加 situ 版才能开这条 decode 快路)。

### P1 — KDA / MLA(95 层的注意力主体)

4. **★★ KDA 在 gfx950 上原生化 / 调优**
   现状:71/95 层的 KDA 递归核 100% 靠**外部 fla**(triton),在 gfx950 上未必调优,且曾被列为数值可疑点。
   要做:评估 fla 在 gfx950 的性能;若差,做 aiter/自研 KDA kernel(chunk + recurrent),
   并把 **causal_conv1d 融进 KDA 前处理**(现在是独立 atom triton kernel)。这是单块最大的连续计算。

5. **★ MLA prefill 从 torch SDPA 迁到 aiter**
   现状:MLA prefill 走 `_prefill_sdpa`(torch),只有 decode 走 aiter unified_attention。
   要做:修好 unified_attention 的 192 varlen prefill fault 后,prefill 也走 aiter,省一条 torch 路径。

6. **★ MLA 192:FLASH vs SHUFFLE 布局**
   现状:因 aiter `unified_attention(shuffled_kv_cache=True)` 在 head_dim=192 误读,gfx950 被迫用 FLASH 4-D 布局。
   要做:修 aiter 的 192 SHUFFLE 5-D 读取;若 SHUFFLE 布局 decode 更快/更省带宽,可切回。

### P2 — 投影融合 / 小算子融合

7. **★ KDA in-proj 融合**:`q/k/v_proj`(+`b_proj`/`g_proj`)现为多个独立 GEMM;
   参考 Qwen GDN 的 `in_proj_qkvz(ba)` 融成一把大 GEMM,减少 kernel launch 与量化重复。

8. **★ MLA a-proj 融合**:`q_a_proj` 与 `kv_a_proj_with_mqa` 都从 hidden 出发,可融;`q_b`/`kv_b` 同理。

9. **KDA o_norm 用融合 gated RMSNorm**:`KimiRMSNormGated` 现为 torch;aiter 有 `gated_rmsnorm*` 可替换。

10. **Dense/shared SiTU 融合激活**:`SituAndMul` 现为 torch;暴露一个 aiter/triton `situ_and_mul`(类比 `silu_and_mul`)。

11. **`_apply_attn_res` 融合**:每层 2 次、~95 层的 torch(RMS 归一+打分+softmax+加权和),可融成一个小 kernel。

12. **AllReduce+RMSNorm 融合覆盖**:确认残差归一点(尤其 latent MoE all-reduce 后 → 下层 input_layernorm)都走
    aiter `tensor_model_parallel_fused_allreduce_rmsnorm(_quant)`,减少 all-reduce/HBM 往返。
