# Kimi-K3 on ATOM — Model Structure, Kernels & Optimization Notes

> Kimi-K3 是一个 **多模态 + 逐层混合注意力(KDA 线性注意力 + MLA 全注意力)的 MoE** 大模型。
> ATOM 只服务其文本路径(权重在 checkpoint 的 `language_model.*` 下)。本文汇总 K3 的模型结构、
> 各算子的 kernel 现状,以及可做的 fusion / 性能优化。
>
> - 模型代码:`atom/models/kimi_k3.py`
> - 参考实现:`modeling_kimi.py`(`KimiDeltaAttention` / `KimiMLAAttention` / `KimiSparseMoeBlock`)
> - aiter:`aiter-k3`(branch `xiaoing/k3`,已加 `ActivationType.Situv2`)

## 1. 模型结构

### 1.1 逐层混合骨架

```
KimiK3ForCausalLM → language_model (KimiLinearForCausalLM)
  ├─ embed_tokens
  ├─ layers[~95]: KimiDecoderLayer      每层二选一注意力 + 二选一 FFN
  │    Attention: layer∈kda_layers → KimiKDAAttention(线性)
  │               else            → KimiFullAttention(MLA)
  │    FFN:       layer≥first_k_dense_replace 且 % moe_layer_freq==0 → KimiSparseMoeBlock
  │               else → KimiMLP(dense, layer0)
  ├─ norm (RMSNorm) + [output_attn_res_*]
  └─ lm_head
```

### 1.2 KDA 线性注意力(`KimiKDAAttention`,71 层)

**门控 delta-rule 线性注意力**(Gated DeltaNet 家族的细粒度门变体)。从 hidden `x` 出发 5 路进核 + 1 路输出门:

```
q_proj, k_proj, v_proj ─cat→ causal_conv1d(+silu) ─→ q,k,v      (读写 conv_state)
f_a_proj→f_b_proj → gate g          b_proj → beta          g_proj → 输出门
```

递归核 `_run_kda`(外部 `fla.ops.kda`),对每 token 维护一块 `S[dv,dk]` 关联记忆(= `ssm_state`):

```
① q,k = L2norm(q), L2norm(k)                 (use_qk_l2norm_in_kernel)
② alpha = 由 g / A_log / dt_bias 合成的逐通道遗忘量
③ b     = sigmoid(beta)  写入强度
④ 遗忘:  S = S * alpha                       (逐 key 通道差异化淡化)
⑤ 读旧:  v_old = S @ k
⑥ 误差:  e = v - v_old
⑦ 更新:  S = S + b * outer(e, k)             (delta:擦旧写新,而非盲目累加)
⑧ 输出:  o = S @ q
```
输出经 `KimiRMSNormGated(o, g_proj(x))` + `o_proj`。**cache 定长、O(1),不随序列增长。**
- `conv_state`(短期滑窗,bf16)·`ssm_state`(长期记忆 S,**fp32**)。
- vs Qwen GDN:GDN 是**每头标量门**,KDA 是**每头每通道细粒度门** + 门下界(`gate_lower_bound`)。

### 1.3 MLA 全注意力(`KimiFullAttention`,24 层)

DeepSeek-MLA 的低秩投影(q_a/q_b/kv_a/kv_b LoRA),但 **`use_mla=False`,把 V padding 到 `Hq=192` 存进标准 paged-MHA cache**,复用现成 MHA kernel。特点:
- MQA 式 k_rope(单头 rope key 扩到所有头);RoPE 仅作用在 64 维 qk_rope 上。
- **输出 sigmoid 门**:`attn_out *= sigmoid(g_proj(x))`(标准 MLA 没有)。
- head_dim=192 是非 2 的幂 → 见 §4 的 aiter shuffle 读 bug。

### 1.4 MoE(`KimiSparseMoeBlock`)

**DeepSeek-V3 式路由 + 可选 latent 压缩 + 共享专家 + SiTU 激活**:
- 路由:`gate`→sigmoid,`e_score_correction_bias` 纠偏,grouped top-k(`n_group=1`)。
- **Latent MoE**(`routed_expert_hidden_size` 设置时):专家在压缩隐维 `Rh` 内算,`routed_expert_down/up_proj` 全专家共享 → 省参数。
- **共享专家**:`num_shared_experts` 个共享专家**融合成一个胖 MLP**(`intermediate = Im·n_shared`),永远激活。
- 激活:**SiTU**(见 §1.6)。
- 关键正确性:latent 路径 **all-reduce 在 `routed_expert_norm` 之前**(非线性 norm 不能作用在 TP 部分和上,commit e68fa86)。

### 1.5 Attn-Residual(块残差,`attn_res_block_size=12`)

一种**沿深度方向的动态残差**(类 Hyper-Connections):
- 维护一个**快照栈** `block_residual`,每 12 层压一张当前流的快照;块内普通累加,块边界存快照并重置。
- 每层 attn 前、mlp 前各做一次 `_apply_attn_res`:对 `[所有快照 + 当前流]` 做**逐 token、内容相关的 softmax 加权检索**(RMS 归一 → 学习方向点积打分 → softmax → 加权和),各用独立学习方向。
- 出网络再检索一次(`output_attn_res_*`)。让深层能按需直接取用任意浅层表示。

### 1.6 SiTU 激活(`SituAndMul`)

Kimi 自定义激活 **SiTU = Sigmoid-Tanh Unit**,替代 SiLU/SwiGLU:
```
out = beta·tanh(gate/beta)·sigmoid(gate) · linear_beta·tanh(up/linear_beta)
      (beta=4, linear_beta=25)
```
本质是 **tanh 软钳制的有界 SiLU** + GLU;tanh 软钳制利于低精度(fp8/mxfp4)数值稳定。
`situ ≈ silu` 于小激活,但 tanh 饱和后发散 —— 用 silu 顶替会污染大幅值输出(曾致 gsm8k 掉点)。

## 3. Fusion & 优化机会

按「收益 × 可行性」排序。★ = 建议优先。

### 3.1 aiter kernel 层面

1. **★ KDA 原生化(去 fla 依赖)**
   当前 KDA 递归核 100% 靠外部 `fla`(triton),aiter 只有 Qwen GDN(标量门)。
   做一个 aiter KDA kernel(细粒度门 delta:chunk_kda + fused_recurrent_kda),并把 **conv1d 融进 KDA 前处理**。
   收益:去外部依赖、ROCm 上可深度调优、和 conv/gate 融合;这是 71/95 层的热点,收益最大。

2. **★ MLA head_dim=192 的 SHUFFLE 读 bug 修复**
   `unified_attention(shuffled_kv_cache=True)` 在非 2 幂 192 误读 5-D shuffle KV,现靠 FLASH 4-D 布局绕开。
   修 5-D shuffle 读取后可切回 SHUFFLE 布局(通常 decode 更快 / 更省带宽)。

3. **MLA prefill 上 aiter**
   prefill 现在是 torch SDPA。修好 unified 的 192 varlen fault 后,prefill+decode 统一走 aiter,省一份 SDPA 路径。

4. **gfx950 融合 SiTU**
   gfx1250 已有 flydsl 融合 situv2;gfx950 的 triton MoE 仍把 situ 放 wrapper torch fp32 算(两个 GEMM 间落地一次 HBM)。
   给 gfx950 triton MoE 加 in-kernel situ(参考 gfx1250 的 `situ_elem`/`situ_mul_vec4`)。

5. **a8w4 GGUU decode 的 SiTU 版本**
   `triton_kernel_fused_experts_a8w4_silu_gguu` 硬编码 silu,已 assert 拒绝 situ → K3 用不了这条 decode 快路。
   加 situ 版可开启 decode 快路。

### 3.2 投影融合(减少 GEMM 数 / launch)

6. **KDA in-proj 融合**
   `q_proj/k_proj/v_proj`(+ `b_proj`/`g_proj`)现在是多个独立 GEMM;参考 Qwen GDN 的 `in_proj_qkvz(ba)` 融成一把大 GEMM,减少 kernel launch 与量化重复。

7. **MLA a-proj 融合**
   `q_a_proj` 与 `kv_a_proj_with_mqa` 都从 hidden 出发,可融成一个 GEMM;`q_b`/`kv_b` 同理可并。

8. **shared experts 已是融合胖 MLP**(N 个共享专家 → 1 个宽 MLP),此项已完成。

### 3.3 Norm / 激活 / 残差 融合

9. **KDA o_norm 用 aiter gated RMSNorm**
   `KimiRMSNormGated`(RMS·weight·sigmoid(gate))现在是 torch;aiter 有 `gated_rmsnorm*`,可替换为融合 kernel。

10. **Dense/shared SiTU 用融合激活算子**
    `SituAndMul` 是 torch;可暴露一个 aiter `situ_and_mul`(类比 `silu_and_mul`),dense/shared 路径直接调。

11. **`_apply_attn_res` 融合**
    每层 2 次、~95 层,纯 torch(RMS 归一 + 打分 + softmax + 加权和 over [K+1,H])。可融成一个小 kernel;
    虽单次量小,但层数多、且在关键路径,值得做成一个融合算子。

12. **AllReduce + RMSNorm 融合覆盖检查**
    aiter 有 `tensor_model_parallel_fused_allreduce_rmsnorm(_quant)`;确认 K3 的残差归一点(尤其 latent MoE
    all-reduce 后 + 下一层 input_layernorm)都走了融合版,减少一次 all-reduce/HBM 往返。

### 3.4 系统级

13. **CUDA Graph 覆盖**(已有 MI355 tp8 脚本):确认 KDA(mamba cache 更新)、attn-residual 的动态形状都能被 graph 捕获。
14. **KV cache dtype**:MLA KV 目前 bf16;评估 fp8 KV(head_dim=192)对精度/带宽的取舍。
15. **KDA prefill chunk 大小 / decode 递归的 tuning**(fla 或未来 aiter kernel 的 tile 配置)。
